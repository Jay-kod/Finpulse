# Component 16: Analytics Engine

## Overview
The Analytics Engine aggregates the rich, granular metadata generated by the 3-stage data pipeline (NLP, ML Classification, Sentiment Analysis) into high-level, actionable insights. It serves as the primary data source for the Analyst Dashboard (`/analytics`).

This component replaces previous hardcoded mock data with robust, real-time database queries utilizing Laravel's Eloquent ORM, and visualizes the results using Chart.js.

## Architecture

### Service Layer (`App\Services\AnalyticsService`)
The `AnalyticsService` acts as the single source of truth for dashboard metrics. It strictly queries `Review` records that have completed the full pipeline (`sentiment_status = 'analyzed'`).

- **`getOverviewStats()`**: Calculates total processed reviews, average compound sentiment score, percentage of reviews flagged as bugs, and the single most frequently occurring topic.
- **`getSentimentDistribution()`**: Categorizes compound scores into three buckets (Positive `[>= 0.05]`, Negative `[<= -0.05]`, Neutral `[between]`) and returns counts.
- **`getTopicDistribution()`**: Aggregates the top 5 most discussed topics.
- **`getIntentDistribution()`**: Aggregates the distribution of user intents (e.g., Feature Request, Complaint, Praise).
- **`getSentimentOverTime(int $days)`**: Groups the average compound sentiment score by `DATE(published_at)` to provide a time-series trend over the last N days.
- **`getRecentAnomalies()`**: Fetches the 10 most recent reviews flagged as bugs (`is_bug = true`) for immediate attention.

### Controller & View Integration
- **`App\Http\Controllers\Analyst\AnalyticsController`**: Injects `AnalyticsService`, calls all six data methods, and passes them to the view.
- **Dashboard (`resources/views/analyst/analytics/index.blade.php`)**:
  - Displays the four primary KPIs in top-level cards.
  - Integrates four distinct Chart.js canvases, passing the PHP arrays directly to `data-` attributes via `json_encode()`.
  - Replaces the generic review table with a targeted "Recent Bug Reports" table.

### Frontend (`resources/js/analytics.js`)
Handles the instantiation of Chart.js objects:
1. **Sentiment Trend**: A Line chart mapping average sentiment over time.
2. **Sentiment Split**: A Doughnut chart visualizing the Positive/Neutral/Negative split.
3. **Top Topics**: A Bar chart displaying the 5 most common topics.
4. **User Intents**: A Bar chart displaying intent distributions.
